# 页面内容维护说明

## 文件分工

- `index.html`：四屏页面结构，只在增删模块时修改。
- `content.js`：页面内容源。标题、说明、链接、图片路径、流程步骤都在这里修改。
- `styles.css`：视觉样式与响应式规则。
- `script.js`：把 `content.js` 渲染进页面，并处理导航、进度和锚点定位。

## 日常更新

1. 打开 `content.js`，修改对应字段。
2. 图片或视频放入 `assets/`，下载文件放入 `downloads/`，再更新 `content.js` 中的相对路径。
3. 本地刷新页面检查结果。
4. 提交并推送到部署仓库后，公开网址保持不变，线上内容自动更新。

## 边界

- 只调整文案、链接和素材时，不改 `index.html`。
- 只有新增第五屏、改变模块顺序或新增交互组件时，才需要同步修改结构或脚本。
- 线上网页不会自动读取本地文件；本地修改必须推送并完成部署后才会出现在公开网址。
